Bachelor Project: AR Demo with Google Cardboard based on JavaScript

by Lukas D. Riesen in April 2019

Application Navigator

The following list contains the links to the code and therefore to the applications accordingly described in the documentation.
For reproduction reasons the downloaded JavaScript libraries can be found at ../Source/third-party/

  • Chapter 1: Introduction

    Background and Motivation: Deprecated GitHub Repository of Awe.js
    There are HTML-files located in the sub-directories of the folder examples which act as executable files in the browser. The repository was downloaded from https://github.com/awe-media/awe.js/tree/deprecated

  • Chapter 2: WebGL Rendering Library

    The library JavaScript file three.min.js and the other needed files from https://threejs.org/ are placed in the sub-directory third-party of Chapter2.
    Testing the browser WebGL support on https://get.webgl.org

    Base Code: baseCode.html was created accordingly to the three.js documentation creating a scene.
    can be run directly locally meaning that the files can be called by their path in the browser

    Loading Objects: baseCodeExtendedObject.html used the example webgl_loader_obj as model. Its source can be found on GitHub
    needs to be run on a server

    Loading Textures: baseCodeExtendedTexture.html using TextureLoader
    needs to be run on a server

    Materials and Lights: baseCodeExtendedMaterialAndLight.html added code according to Aerotwist - Getting Started with Three.js
    can be run directly locally

    Manipulating Objects: baseCodeExtendedManipulation.html inspired by webgl_animation_skinning_blending and webgl_animation_skinning_morph
    can be run directly locally

  • Chapter 3: AR Library

    The following application can be run locally, on http://localhost or on a server with HTTPS.
    For the testing it is important to have the marker 64.png printed out and the device has a camera.
    Try to hold the marker in different positions and places to see the limitations...
    however the intended idea for most of the following applications is to hold it like reading a book.
    Loading can take a bit of time.

    Base Code: baseCode.html is based on aw.js marker_ar example from Chapter 1.

    Building Enhanced Scene: enhancedScene.html

    Marker as Controller: markerController.html

  • Chapter 4: Google Cardboard AR demo

    Same preconditions as for chapter 3. Additional a Google Cardboard with a hole for the device camera is needed.

    Fullscreen: fullscreen.html derived from markerController.html from Chapter 3.

    Stereoscopic View: stereoscopicView.html toggle jsartoolkit registration to load split view.
    this example demonstrates the problem.

    Workaround: workaround.html toggle jsartoolkit registration to load split view.
    in this example marker detection works after setting jsartoolkit back to registration.

    Final Demo: finalDemo.html
    Description on how to use it in documentation.
